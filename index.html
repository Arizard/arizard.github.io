RedactedHub

Redactedhub is a multi-purpose server hub which I created for my community. It features a point system, item store (cosmetic and augmenting), jukebox (using youtube videos as the source), a donation link, as well as 2 spare panels in which I hope to add item trading and an admin control panel.

RedactedHub differs from other pointstores due to the way items are stored. Each item is an individual row in a table, and each row has a specified owner. This means that it is possible to have a limited number of units of each item, improving a server's point economy. Once a limit for a certain item is reached, there can be no more sold, thus causing the players to trade the item at a higher value as demand increases. Points are also finite, which means that the store starts off with a certain amount of cash, and as points are given out, cash in the store is depleted - there is a limit where the store can no longer give out points, and the players will need to trade between eachother or purchase more items from the store, causing points to increase in value due to them being harder to obtain.

The jukebox features a playlist (which changes depending on the server you are on) and a queue, which stores songs that you have added from the playlist. If a song is added to your queue, you will still be able to listen to it if you join another server with RedactedHub installed - the queue is stored on your client and remains the same regardless of the server you are on.

bodyGroupr

Too often have we needed to sit down and wait for hundreds of megabytes of playermodels to download. It's a pain, and we're sick of it.

But what if we didn't have to? What if there was an addon that allowed us to unlock the full potential of many workshop playermodel packs?

Bodygroups, if you haven't heard of them by now, are small changes to a playermodel's appearance (such as clothing) which are included within the model's files. Rather than having separate playermodels for each different item of clothing, bodygroups allow you to interchange, switch, swap, and shuffle your playermodel's appearance to any combination you desire.

When using bodyGroupr, these bodygroups are no longer locked away in the code, and can be accessed freely by the players. This way, you (the owner) can minimize the disk space and network bandwidth that gets wasted each time a new player joins!
